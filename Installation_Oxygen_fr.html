1. Inscrivez-vous sur GitHub. GitHub. GitHub est une plateforme pour les développeurs d'outils numériques, spécialisée dans le travail collaboratif. Nous l'utilisons comme un dépôt en nuage qui nous permet d'avoir toujours accès à la version la plus récente des fichiers. Contrairement aux autres dépôts en nuage, vous devez toujours (!) donner la commande "fetch" ou "pull" pour télécharger et synchroniser. Allez sur GitHub et inscrivez-vous avec une adresse e-mail. Vous devez créer un nom d'utilisateur et un mot de passe. Veuillez m'envoyer le nom d'utilisateur et l'adresse e-mail afin que je puisse vous donner accès à notre dépôt; vous en aurez besoin pour le point 9.


2. Créez un dossier sur votre ordinateur dans lequel vous souhaitez conserver les fichiers de ce projet, par exemple Grégoire.


3. Téléchargez et installez GitHub-Desktop. Desktop est une petite console qui vous permet de gérer facilement les fichiers que vous téléchargez et mettez en ligne depuis GitHub. Il est très maniable et utile.


4. Téléchargez et installez la police de caractères Junicode. Junicode est une police de caractères spéciale pour les médiévistes et nous l'utilisons pour les transcriptions. Vous pouvez télécharger cinq polices: Junicode en 4 modes (normal, italique, gras et italique-gras) et FoulisGreek. Cette dernière est facultative, mais les autres sont nécessaires.


5. Téléchargez et installez Oxygen XML-Editor. Installez impérativement la version pour laquelle nous vous avons envoyé la clé de licence. Lors de l'installation, la clé de licence vous sera demandée et vous devez cocher l'option "Utiliser la clé de licence". Copiez dans la fenêtre la clé de licence (uniquement les 9 lignes entre "Début" et "Fin").


6. Vous devez maintenant installer un add-on pour pouvoir utiliser le framework de Gustavo Fernández Riva. Ce framework vous permet de travailler en mode auteur, sans voir les codes et avec un menu spécial pour notre codage.

a) A l'intérieur d'Oxygen, allez à : Help => Install new Add-ons. Dans la fenêtre "Show new Add-ons from", copiez l'adresse suivante: https://digi.ub.uni-heidelberg.de/frameworks/IweinFramework.xml.

b) Cochez la case du framework heiEditions et suivez toutes les étapes de l'installation. A la fin, vous devez fermer Oxygen et le relancer.


7. Vous devriez maintenant également procéder à quelques réglages dans Oxygen. Allez dans le menu supérieur sur Options => Préférences.

a) Dans le menu de gauche, allez à l'entrée "Editor" et cliquez sur "Format". Dans l'entrée "Line Width", réglez le chiffre "10000".

b) Allez maintenant dans le sous-menu "Format" sur le côté gauche et cliquez sur XML. Sur le côté droit, allez dans la section "Elements spacing". Cliquez sur l'onglet "Preserve Space", cliquez sur le bouton "Add" en bas et ajoutez "l//*" (un l comme 'ligne', sans les guillemets).

c) Maintenant, allez dans l'onglet "Mixed Content" et ajoutez "l" (un l comme 'ligne') de la même manière.

d) Dans l'onglet "Line Break", ajoutez également "l".

e) Cliquez sur Ok et fermez.


8. Faites maintenant également quelques réglages dans le framework.

a) Ouvrez un fichier de modèle et cliquez en bas au milieu de la fenêtre sur le bouton "Author".

b) Cherchez parmi les boutons de fonction en haut celui qui s'appelle "Styles" et déployez-le. Activez "Nur Abkürzung" et désactivez "Nur Auflösung". Plus tard, vous pourrez modifier cela à votre guise.

c) Cherchez maintenant une ligne plus bas un bouton avec trois triangles jaunes et dépliez-le. La plupart des gens travaillent avec des "partial tags", mais vous pouvez aussi essayer avec des "full tags".


9. Quand je vous ai donné les droits d'accès à GitHub (donc un autre jour), allez sur GitHub Desktop et choisissez "File" => "Clone repository".

a) Dans la fenêtre qui s'ouvre, cliquez sur l'onglet URL;

b) Dans "Repository URL", copiez le lien suivant: https://github.com/v-millet/Gregoire.git.

c) Dans "Local Path", cliquez sur "choose" et sélectionnez le dossier sur votre ordinateur que vous avez configuré au point 2 ci-dessus.

d) Cliquez maintenant sur "Clone" et tous les fichiers et dossiers qui se trouvent sur le repository seront copiés sur votre ordinateur.


10. A partir de ce moment, vous pouvez travailler. Si vous apportez des modifications dans l'archive, les autres collaborateurs du projet peuvent les voir et les corriger. Si ces derniers effectuent des modifications, elles vous seront indiquées.


11. L'habitude de travail suivante est importante:

a) Avant de travailler, ouvrez toujours le Desktop, cliquez sur "Fetch" (et ensuite sur "Pull" s'il y a quelque chose à mettre à jour). Ce n'est qu'ensuite que vous ouvrez votre fichier.

b) Lorsque vous avez terminé, enregistrez le fichier sur votre ordinateur et allez ensuite toujours a GitHub-Desktop; vous verrez que le fichier apparaît à gauche et les modifications que vous avez apportées à droite. En bas à gauche, sous "Summary", indiquez ce que vous avez fait (par exemple "jusqu'à v. 123" ou "Raccourcis"), puis cliquez sur "Commit" et ensuite sur "Push" en haut à droite. Vous mettez ainsi à jour le fichier sur GitHub.

c) Encore une fois: toujours deux clics, aussi bien pour télécharger que pour charger des fichiers.
